# External Validation: What to do now

## Important

Do not invent values. The final `validation_data/external_reference.csv` must contain values generated outside the ISRS_SCL_Q2 model, such as:

- GNPy reference simulation
- SSFM / split-step Fourier simulation
- laboratory or vendor measurement

A fast preview/debug run can skip external validation, but a publication run must use real independent values.

## Files in this pack

- `validation_data/external_reference.csv`
  - Header-only final file. Fill it with real rows before publication mode.
- `validation_data/external_reference_measurement_plan.csv`
  - Ready-to-fill measurement plan covering S/C/L, span counts, flat/fixed/adaptive, and GSNR/NLI metrics.
- `tools/check_external_reference_schema.py`
  - Local checker for missing columns, blank metric values, blank provenance, and invalid independence labels.

## Required columns

source_id:
  Unique ID for each reference row.

source_type:
  `GNPy`, `SSFM`, `Experiment`, or another clearly independent source.

tool_version:
  Exact tool version, Git commit, script hash, or instrument/software version.

configuration_hash:
  Hash of the configuration used to generate the reference. Use the resolved config hash from RUN_MANIFEST.json if available,
  or record SHA-256 of the reference simulator config.

date:
  YYYY-MM-DD.

independent:
  Must be `true` for publication evidence. If the values come from your own model output, it is not independent.

provenance_reference:
  Path/identifier for the raw output file, lab notebook, script, or simulator config used to produce the row.

span_count:
  Number of 80-km spans. Use the same physical distance assumptions as the model being validated.

wavelength_nm:
  Channel wavelength being compared.

band:
  S, C, or L.

strategy:
  flat, fixed, or adaptive.

metric:
  Recommended minimum: gsnr_db. Add nli_w if your external source can report it.

metric_value:
  The actual numerical value from the independent source.

metric_unit:
  dB for gsnr_db, W for nli_w.

uncertainty:
  Uncertainty of the reference value. If unknown, use a justified estimate and explain in notes.

notes:
  Any details needed to reproduce the row.

## Fast practical path with GNPy

1. Install GNPy in a separate folder/environment so it is independent of your ISRS_SCL code.
2. Build equivalent GNPy network/config files for your test cases:
   - 80 km span length
   - same channel spacing
   - matching fiber attenuation/dispersion/gamma assumptions where possible
   - matching amplifier noise figure and gain settings where possible
3. Run GNPy for selected S, C, and L wavelengths, multiple span counts, and strategies.
4. Copy GNPy output GSNR values into `metric_value`.
5. Put the raw GNPy output files under a provenance folder such as:
   `external_validation/gnpy/raw_outputs/`
6. Keep `independent=true` only if the values were generated by GNPy/SSFM/measurement, not copied from ISRS_SCL outputs.
7. Run:
   `python tools/check_external_reference_schema.py validation_data/external_reference.csv`
8. Then rerun:
   `python run_all_experiments.py --debug --config config_q2_final.yaml --run-id q2-debug-external-001 --output-root runs --overwrite`

## Minimum recommended coverage

For a strong Q2 claim, use at least:
- S, C, and L bands
- 3 wavelengths per band
- at least 2 span counts, preferably 1, 4, and 8
- flat and fixed strategies; adaptive if you can reproduce the same launch profile externally
- at least one independent source; two is stronger, e.g., GNPy + SSFM
